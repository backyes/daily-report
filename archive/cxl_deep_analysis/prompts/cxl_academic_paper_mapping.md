## 分析框架

### 研究主线 1：混合精度 KV 量化与组合缓存

#### 代表论文方向
- 2-bit/4-bit 敏感度感知 KV 量化（MLSys 2024-2026）
- Channel-wise Boost Caching（ASPLOS 2025-2026）
- Bit-serial PE 优化总线吞吐（ISCA 2024-2026）

#### 映射到 5 指标矩阵
- **有效带宽**：量化将 KV Cache 大小减少 4-8× → 等效带宽提升 4-8×
- **延迟**：反量化引入 ~1-5μs 延迟 → 需要预取隐藏
- **单位成本**：容量需求降低 → CXL-DDR 可能替代部分 HBM → 成本降低 2-3×
- **耐久度**：量化后的写入量减少 → Flash 耐久度需求降低

#### 关键量化数字（从论文提取）
- 量化位宽 vs 精度损失的帕累托前沿
- 反量化延迟的绝对值（μs）
- 端到端吞吐影响百分比

### 研究主线 2：异构分层内存迁移

#### 代表论文方向
- 多级内存管理（HBM → CXL-DDR → HBF → 并行文件系统）（ISCA 2024-2026）
- 预测贝叶斯重用算法（HPCA 2025-2026）
- 对象重聚合优化 NUMA 页召回（ASPLOS 2024-2026）

#### 映射到 5 指标矩阵
- **IOPS**：预测算法减少冷数据无效访问 → 有效 IOPS 提升
- **延迟**：预取将 CXL/HBF 延迟从关键路径移除 → 感知延迟接近 HBM
- **有效带宽**：对象重聚合将随机小 IO 转为顺序大 IO → 带宽利用率从 20-30% 提升至 80-90%

#### 关键量化数字（从论文提取）
- 预测命中率（如 85-95%）
- 预取距离（多少个 token 提前）
- 页迁移开销（μs 级）
- 端到端延迟改善百分比

### 研究主线 3：近存计算 / PIM 卸载

#### 代表论文方向
- 注意力矩阵池化卸载到内存控制器（ISCA 2024-2026）
- 稀疏索引聚合 PIM（HPCA 2025-2026）
- KV 解压缩卸载到 CXL 控制器（MLSys 2025-2026）

#### 映射到 5 指标矩阵
- **有效带宽**：数据移动量减少 50-90% → CXL 链路利用率大幅提升
- **延迟**：卸载计算在控制器侧完成 → 减少 GPU-CXL 往返
- **IOPS**：部分 IOPS 被控制器吸收 → 减少 CXL 链路上的 IOPS 压力

#### 关键量化数字（从论文提取）
- 卸载前后的 CXL 数据移动量对比
- 控制器侧计算延迟（μs）
- GPU 利用率提升百分比
- 端到端吞吐提升百分比

### 研究主线 4：稀疏注意力与内存优化

#### 代表论文方向
- Paged-sparsity 框架（MLSys 2025-2026）
- CXL NUMA-hiding 引擎（HPCA 2025-2026）
- 预测性 KV Cache 驱逐策略（ASPLOS 2025-2026）

#### 映射到 5 指标矩阵
- **IOPS**：稀疏注意力减少 KV 页访问量 → IOPS 需求降低
- **延迟**：NUMA-hiding 引擎将 CXL 延迟屏蔽 → 端到端延迟不增加
- **容量**：驱逐策略减少 KV Cache 常驻容量 → 同等容量支持更长上下文
